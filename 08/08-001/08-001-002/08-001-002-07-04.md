# 面向对象的三大特性之一：继承 06,08-12 视频day07 Python文献

- 子类（派生类）可以继承父类（基类）所有的非私有成员。
    - 实现了代码复用。
        - 包含new方法和init方法等。
        - 由于存在名称改写，子类不可以继承父类的私有成员。
    - 子类可以新增自己的成员。
    - 子类可以重写父类的方法。
- 继承：`class 子类(父类1[, ...]):`
- 在子类中调用“父类”的方法：`super().方法()` #gemini-l
    - `super()`：等价于`super(type, obj)`。
        - `type`：定义`super()`所在方法的类对象。
        - `obj`：调用`super()`所在方法的类对象或实例对象。
            - 每一个类都有一个内置属性`__mro__`（元组），存储了该类在查找方法时的线性化顺序。
                - 必须通过类对象访问。 #todo 属性查找顺序
                - 方法解析顺序 (Method Resolution Order, MRO) 的三大原则：
                    - 子类总是优先于父类。
                    - 如果为多继承，左边的类优先于右边的类。
                    - 子类的MRO不可以违背父类们既有的MRO相对顺序，否则引发`TypeError: Cannot create a consistent method resolution`。
            - 在调用“父类”的类方法和实例方法时，会自动传入`cls`和`self`。
    - `super().方法()`：`super(type, obj).方法()`：
        1. 通过`obj.__mro__`或`obj.__class__.__mro__`获取MRO。
        2. 在MRO中找到*`type`的下一个类——查找方法的起点*。
        3. 从该类开始（包含该类）向后/链式查找方法，找到就调用。
    - 一般情况下，统一使用`super()`，可维护性和扩展性较好。
        - 不使用`super(type, obj)`和`父类.方法()`，硬编码。
        - 不使用`self.方法()`，如果存在方法的重写，那么`self.方法()`调用的是子类重写的方法。
    - 如果为多继承：
        - 链条上的每一个类的init方法都必须显式调用`super().__init__(*args, **kwargs)`。
            - 根基类（直接父类为`object`）的init方法，必须耗尽`args`和`kwargs`。
                - `object`的init方法只接收`self`，否则引发`TypeError: object.__init__() takes exactly one argument (the instance to initialize)`。
            - 避免MRO链条断裂。
        - init方法的签名：`__init__(self, *args, **kwargs)`。
            - 无法预知当前MRO链条中的下一个类，定义具体的形参极易导致参数解包失败。
            - `实例属性 = kwargs.pop('实例属性名')` -> `super().__init__(*args, **kwargs)`
- 方法的重写：子类重新实现父类的方法（方法名必须相同，形参列表可以不同）。
    - **类对象的命名空间是字典——方法名和属性名都是键。**
        - => Python没有方法的重载（后面定义的方法对象会覆盖掉前面定义的同名方法对象）。
        - => 方法的查找*只看方法名不看参数*（按照MRO，根据方法名访问方法对象）

---

- init方法的参数是可变长的。
- `object`是所有类的直接父类或间接父类。
