# 异常处理 05-08 视频day08 Python文献

- 异常处理：
    - 不是规避错误，而是发生错误时，提供处理方案，以让程序继续执行。
    - 如果没有处理异常，则不会执行异常发生后的代码。

```python
try:
    可能发生异常的代码
except 异常类1 as 变量名:
    处理该异常的代码
...
except (异常类2, 异常类3, ...) as 变量名:
    处理该异常的代码
...
except:
    处理该异常的代码
else:
    try块没有发生异常，希望执行的代码
finally:
    无论如何，都会执行的代码
```

- `try`：
    - 如果发生异常：
        - 创建并抛出对应的异常类对象。
            - 如果异常类对象没有被任何`except`捕获，则向外抛出。
                - 异常传递顺序 #todo 方法内发生异常，抛出异常类对象给方法调用者。
        - 不会执行`try`块中异常发生后的代码。
- `except`：类似`if-else`分支，对不同类型的异常进行不同的处理：
    - `except 异常类1 as 变量名:`：捕获`异常类1`的对象，并赋值给`变量名`。
    - `except (异常类2, 异常类3, ...) as 变量名:`：捕获在元组中的异常类的对象，并赋值给`变量名`。
    - `except:`：捕获其他所有异常类的对象。
        - 位置：最后一个`except`。
    - `print(异常类的对象)`：打印该异常类的描述信息。
- `else`（用得不多）：
    - 位置：在所有`except`之后。
    - 从执行效果上来说，将“不会发生异常的代码”放在`else`块和放在`try`块是等价的。
        - 逻辑区分“可能发生异常的代码”和“不会发生异常的代码”。
- `finally`：
    - 位置：在所有`except`或`else`之后。
    - 无论如何（除非`finally`块发生异常）：
        - `try`块、`except`块、`else`块，无论是否发生异常。
        - `try`块抛出的异常类对象，无论是否被`except`捕获。
    - 从执行效果上来说，如果`try-except`结构正常执行完毕，将代码放在`finally`块和放在`try-except`结构外是等价的。
        - 放在`try-except`结构外的代码，只会在`try-except`结构正常执行完毕后，才会执行：
            - `try`块没有发生异常，且`else`块也没有发生异常。
            - `try`块发生异常，被`except`捕获，且该`except`块也没有发生异常。
    - 一般用于执行某些必须要进行的清理操作：
        - 关闭文件
        - 释放资源（例如：网络连接、数据库连接、锁等）
