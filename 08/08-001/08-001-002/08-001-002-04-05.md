- 函数：封装代码块（实现某个功能），可重复使用。
    - 提高代码的复用性，避免代码冗余。
    - 降低维护成本。
    - 提高可读性。
    - 便于分工协作开发。
- 内建函数（Python提供）、用户自定义函数。
- 定义函数：
    - 参数：
        - 在调用函数时，执行`形参 = 实参`。
            - 形式参数/形参：在定义函数时，定义者声明的参数。
            - 实际参数/实参：在调用函数时，调用者传递的参数。
        - 形参形式：
            - 位置参数 (positional argument)：实参根据形参位置匹配形参。
                - 定义：`位置参数`
                    - `形参, ..., /`：`/`前面的形参强制使用位置参数。
                - 调用：`实参`
                    - 实参个数 == 形参个数 and 实参顺序 == 形参顺序
                    - 如果有多个位置参数，可以对容器解包传参：`*容器`。
                        - `*字典`：等价于`*字典.keys()`。
            - 关键字参数 (keyword argument)：实参根据形参名匹配形参。
                - 定义：`关键字参数`（同位置参数）
                    - `*, 形参, ...`：`*`后面的形参强制使用关键字参数。
                - 调用：*`关键字参数=实参`*
                    - 关键字参数之间无所谓顺序。
                    - 如果有多个关键字参数，可以对字典解包传参：`**字典`。
                        - 键名必须和形参名一致。
            - 默认值参数：
                - 定义：*`位置参数=默认值`*或*`关键字参数=默认值`*
                    - 顺序：*非默认值参数 > 默认值参数*
                    - 根据`默认值`可以反推形参的类型（弱约束）。
                - 调用：`实参`或`关键字参数=实参`
                    - 可省略实参：`默认值`。
                    - 传递实参：`实参`。
            - 可变位置参数 (var-positional argument)：
                - 定义：`*可变位置参数`
                - 调用：`实参1, ...`或`*容器`
                    - 使用*元组*接收*未命名*的实参：`(实参1, ...)`。
                        - 在编写函数体时，可变位置参数的类型就是元组。
                    - 可省略实参：`()`。
            - 可变关键字参数 (var-keyword argument)：
                - 定义：`**可变关键字参数`
                    - 顺序：*位置参数 > 可变位置参数 > 关键字参数 > 可变关键字参数*
                - 调用：`参数名1=实参1, ...`或`**字典`
                    - 使用*字典*接收*命名*的实参：`{"参数名1": 实参1, ...}`。
                        - 在编写函数体时，可变关键字参数的类型就是字典。
                    - 可省略实参：`{}`。
    - 函数的注释（了解）：
        - Python3新增。
        - 注释形参的类型，则可以在PyCharm中通过`.`调用类型的方法。
        - 注释返回值的类型。
        - 通过函数对象的属性`__annotations__` (`dict`) 查看注释。
        - 注释没有约束性，需要在函数内手动检查。
    - `""" 函数说明文档 """`
        - 函数中的第一行。
        - `help(函数名)`：打印函数说明文档。
            - `函数名`不带括号，否则表示调用函数。
    - `return [表达式]`：
        - 给调用者返回结果。
            - `return`：等价于`return None`。
            - 可省略`return`：自动在函数中的最后一行隐式添加`return None`。
            - `return 结果1, ...`：使用*元组*返回多个结果，等价于`return (结果1, ...)`。
        - 结束当前函数的执行。

```python
def 函数名([形参列表]):
    ["""
        函数说明文档
    """]
    函数体
    [return [表达式]]

def 函数名(形参:类型[=默认值], ...) -> 返回值类型:
    ...
```

- 执行函数：
    - 执行（并非调用）某个函数时，该函数必须提前定义。
        - 模块的代码结构模板： #gemini-l
            1. 导包
            2. 声明全局常量
            3. 定义类
            4. 定义业务逻辑函数
            5. 定义主逻辑函数：`def main():`
            6. 入口检查点：
                - `if __name__ == "__main__":`[^1]
                    - `main()`
    - 内存模型： #gemini-l
        - 堆内存：存储对象。
            - 模块对象 (`PyModuleObject`)：
                - *`__dict__`* (*`dict`*)：命名空间，维护模块的成员。
                    - `"成员的名字": 成员的内存地址`
                    - 模块的成员：*全局变量*、函数。
            - 函数对象（`PyFunctionObject`，*定义时创建* => 函数对象可以作为函数的参数或者返回值——面向函数式编程）：
                - `__code__` (`PyCodeObject`)：
                    - `co_varnames` (`tuple`)：存储未被内层函数引用的局部变量名。
                        - 赋值即定义：在函数内对一个没有特别声明的变量进行*赋值*时，会把这个变量定义为局部变量。
                            - 例如`lst[0] = 1`、`lst[:] = [1, 2]`等并非对变量进行赋值。
                    - `co_cellvars` (`tuple`)：存储被内层函数引用的局部变量名。
                    - `co_freevars` (`tuple`)：存储从外层函数捕获的变量名。
                    - `co_code`：字节码。
                        - 根据上面的三个元组，给每个变量分配localsplus的索引 => “明牌”根据索引访问变量。
                            - “根据索引访问元素”比“根据键访问值”快得多，因此CPython主要依赖Fast Locals而非`f_locals`。
                            - 同理，访问局部变量比访问全局变量快得多，因此“在循环内部，如果需要频繁使用全局变量，应先将其赋值给局部变量”。 #todo
                    - ……
                - `__globals__`：指向所属模块的`__dict__`。
                    - 可以访问模块对象的`__dict__`。
                    - `global 全局变量`：不定义为局部变量，一般修改不可变的全局变量的指向。
                - `__closure__` (`tuple`)：存储从外层函数捕获的变量对应的Cell对象。
                    - 从外层函数的栈帧对象的Cell Vars区域传递`PyCellObject`。
                    - `nonlocal 变量`：不定义为局部变量，一般修改不可变的变量的指向。
                    - 延长从外层函数捕获的变量的生命周期。
                - ……
        - 调用栈：存储栈帧。
            - 栈帧（`PyFrameObject`，开始执行时创建并压栈，执行结束后弹栈并销毁）：
                - `f_code`：指向函数对象的`__code__`。
                - localsplus（*指针数组*）：
                    - *Fast Locals*：存储未被内层函数引用的*局部变量*。
                    - Cell Vars：存储被内层函数引用的局部变量对应的Cell对象。
                        - 为`co_cellvars`中的每个局部变量创建一个`PyCellObject`。
                        - `PyCellObject`：
                            - `ob_ref`：局部变量指向的对象的内存地址。
                    - Free Vars：存储从外层函数捕获的变量对应的Cell对象。
                        - 从函数对象的`__closure__`传递`PyCellObject`。
                    - Evaluation Stack：虚拟机执行字节码时的计算栈。
                - `f_locals` (`dict`)：“惰性”属性，显式调用`locals()`，临时获取一个同步Fast Locals的字典。
                - `f_globals`：指向函数对象的`__globals__`。
                - `f_back`：指向调用当前函数的上一个栈帧。
- 在模块中定义函数，在类中定义方法。
- 闭包：
    - 函数嵌套：在外层函数内定义内层函数。
    - 内层函数访问外层函数的局部变量。
    - 外层函数返回内层函数的函数对象。

```python
# 需求：在一个函数内，访问另一个函数的局部变量。

# 方式一：函数嵌套调用
def func1():
    a, b, c = 10, 20, 30
    a, b, c = func2(a, b, c)

def func2(a, b, c):
    print(a, b, c)
    ...
    return a, b, c

 # 方式二：函数嵌套定义：闭包
def outer():
    a = 10

    def inner():
        nonlocal a
        print(a)
        a += 1
        print(a)

    # inner()
    # print(a)

    # 没有括号，表示返回函数对象。
    return inner

# outer()()
f_inner = outer()
f_inner()
```
